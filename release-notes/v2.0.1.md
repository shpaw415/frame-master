# Release Notes - v2.0.1

**Release Date:** November 1, 2025

## Overview

Version 2.0.1 brings significant improvements to the developer experience with enhanced CLI interface, better server initialization, and optimized plugin configuration.

## ‚ú® New Features

### Enhanced CLI Interface

- **Beautiful Terminal Output**: Added colorful, user-friendly CLI messages using chalk

  - Server startup now displays a styled information box with URL and environment mode
  - Error messages are formatted with clear instructions and examples
  - Improved visual hierarchy and readability

- **Better Error Messages**: NODE_ENV validation errors now provide:
  - Clear warning indicators with emoji
  - Multiple solution options (inline command or .env file)
  - Copy-paste ready examples for both development and production modes

### Improved Server Initialization

- **Optimized Plugin Loading**: Enhanced initialization sequence for server configs and plugins
- **Better Configuration Management**: Streamlined server startup process

### Enhanced Build Pipeline

- **Concurrent Build Protection**: The builder now prevents simultaneous builds with clear error messaging, avoiding race conditions and ensuring build integrity
- **Build Status Monitoring**: New public methods for tracking build state:
  - `isBuilding()`: Check if a build is currently in progress before starting new operations
  - `awaitBuildFinish()`: Get a promise that resolves when the current build completes, enabling coordination between dependent tasks
- **Better Build Orchestration**: Improved ability to coordinate multiple processes that depend on build completion, especially useful in watch mode and plugin development
